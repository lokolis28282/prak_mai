Нужно сделать проект для практики МАИ.

Тема:
«Разработка программного обеспечения для автоматизированного учета оборудования на складе ЦОД»

Контекст:
Практика проходит на рабочем месте в ЦОД X5 Tech.
Реальные данные компании использовать нельзя: серийные номера, инвентарные номера, реальные названия оборудования и внутренние данные должны быть заменены на тестовые/обезличенные.

Что нужно сделать:
1. Разработать программу на Python для автоматизированного учета оборудования на складе.
2. Использовать SQLite как локальную базу данных.
3. Сделать тестовые данные, которые имитируют склад ЦОД.
4. Подготовить отчет по практике, где раскрыты пункты руководителя.

Функции программы:
- добавление оборудования;
- учет прихода оборудования;
- учет расхода/выдачи оборудования;
- учет перемещения;
- просмотр остатков;
- поиск и фильтрация оборудования;
- журнал операций;
- экспорт отчета в CSV или Excel;
- работа только с тестовыми/обезличенными данными.

Минимальная структура данных:
- оборудование: id, категория, модель, тестовый серийный номер, тестовый инвентарный номер, статус, местоположение;
- операции: id, дата, тип операции, оборудование, количество, основание, ответственный;
- категории оборудования;
- складские зоны/места хранения.

Отчет по практике должен раскрывать:
1. Анализ бизнес-процессов функционирования склада:
   - показатели эффективности;
   - контекстная диаграмма процесса;
   - проблемные места;
   - функциональные требования к системе.
2. Описание текущего процесса учета оборудования.
3. Разработку ПО для автоматизированного учета.
4. Методику использования системы на предприятии.
5. Анализ эффективности использования разработанного ПО.

Итоговая папка проекта должна содержать:
- исходный код программы;
- SQLite-базу с тестовыми данными;
- README.md с инструкцией запуска;
- requirements.txt;
- отчет по практике в формате Markdown;
- папку docs со схемами/диаграммами;
- папку screenshots или описание, какие скриншоты нужно сделать.

Важно:
- не использовать реальные данные компании;
- все данные должны быть тестовыми;
- проект должен запускаться локально;
- интерфейс можно сделать консольным, главное — чтобы программа работала и закрывала требования практики.
