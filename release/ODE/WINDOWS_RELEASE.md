# ODE: запуск и сохранность данных на Windows

## Установка и запуск

1. Установите Python 3.10 или новее с сайта python.org. При установке включите `Add Python to PATH`.
2. Распакуйте папку `ODE` из архива в постоянное место, например `C:\ODE`. Не запускайте программу прямо из ZIP и не держите её в папке временных загрузок.
3. Дважды щёлкните `start_windows.bat`. Откроется браузер с ODE. Если браузер не открылся, перейдите на `http://127.0.0.1:8765`.

Команда ручного запуска из PowerShell или cmd:

```bat
cd /d C:\ODE
py -3 app.py
```

## Вход и профиль

При обычном входе укажите ФИО инженера. Оно будет записано как автор операций и журнала действий. ФИО и должность показаны справа в шапке; кнопка «Выйти» позволяет сменить инженера.

Старая система пользователей сохранена. Для управления пользователями, резервными копиями и паролем на экране входа выберите «Режим администратора», затем войдите своей администраторской учётной записью. Смена пароля находится в административном режиме. Пароль пользователя `lokolis` при сборке не изменяется.

## База и резервные копии

Рабочая база находится в `C:\ODE\data\warehouse.db`. Все новые операции записываются в этот файл.

Перед backup остановите ODE, затем скопируйте `data\warehouse.db` на другой диск или в защищённую сетевую папку. Имя копии должно содержать дату, например `warehouse_2026-07-05.db`. В административном режиме также доступна кнопка создания резервной копии; такие копии находятся в `data\backups`.

Для восстановления:

1. Остановите ODE.
2. Сохраните текущий `data\warehouse.db` отдельно.
3. Скопируйте выбранную исправную копию на место `data\warehouse.db`.
4. Запустите ODE и проверьте баланс и последние операции.

В административном режиме можно восстановить копию через интерфейс — перед заменой система создаёт страховочную копию автоматически.

## Как не потерять данные

- Не удаляйте и не заменяйте `data\warehouse.db` при обновлении программы.
- Не распаковывайте новый релиз поверх рабочей папки без отдельной копии базы.
- Делайте backup после каждого рабочего дня и храните хотя бы одну копию вне ноутбука.
- Перед выключением дождитесь сообщения об успешной операции и закройте окно запуска обычным способом.
- Периодически запускайте проверку базы в административном режиме; результат должен быть `ok`.
