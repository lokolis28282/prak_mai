# ODE 0.15.0 — приватный перенос на рабочий Windows-ноутбук

Этот комплект содержит внутренние складские данные, реальные S/N и локальные
Monitoring JSON. Не загружать архив в GitHub, публичное облако или личный
файлообменник. Передавать только разрешённым корпоративным способом.

## Распаковка

1. Скопировать архив на рабочий ноутбук.
2. Полностью распаковать, например в `C:\ODE`.
3. Не запускать программу прямо из ZIP и не размещать каталог в OneDrive.
4. Убедиться, что существует `C:\ODE\data\warehouse.db`.

## Проверка рабочей БД

В `cmd.exe`:

```bat
cd /d C:\ODE
certutil -hashfile data\warehouse.db SHA256
```

Ожидаемый SHA-256:

```text
68f06d7a764ac8d2ccde1b59d99ad7977cb665808602d2980a3dfdc87c4a5314
```

## Запуск

1. Установить Python 3.10 или новее с опцией `Add Python to PATH`.
2. Дважды щёлкнуть `start_windows.bat`.
3. Если браузер не открылся, перейти на `http://127.0.0.1:8765`.

Основной ODE не требует внешних Python-зависимостей. Selenium из
`requirements-monitoring.txt` нужен только для optional live DCIM collection и
не требуется для демонстрации Warehouse.

## Безопасная демонстрация

- для просмотра истории и остатков можно использовать перенесённую рабочую БД;
- тестовые приход/расход выполнять только на копии или тестовом контуре;
- перед реальными операциями создать backup в разделе `Администрирование`;
- не заменять `data\warehouse.db`, пока ODE запущен;
- после работы остановить сервер комбинацией `Ctrl+C` в его окне.
