# Master Product Context

ODE — рабочая система учёта оборудования и складских операций для инженеров
ЦОД. Учебная тема МАИ остаётся происхождением проекта, но продукт проектируется
для реальной эксплуатации.

## Цель

Оператор должен безопасно выполнять инвентаризацию, приход, расход,
перемещение, поиск по S/N и Inventory Number, видеть баланс и историю, а
администратор — управлять пользователями, справочниками, аудитом и backup.

Путь эксплуатации:

`MacBook development → локальный пилот → несколько инженеров → сервер X5 Tech → интеграции`.

Серверный режим ещё не реализован. До него ODE остаётся локальным модульным
монолитом Python + SQLite без преждевременных микросервисов.

## Неизменные продуктовые границы

- S/N — основной бизнес-идентификатор serialized equipment.
- Inventory Number может быть назначен существующей карточке позже.
- Raw identifiers и leading zeros не преобразуются в числа и не угадываются.
- Проведённые складские операции не редактируются задним числом без
  утверждённого correction contract.
- Monitoring не пишет Warehouse facts и использует только application ports.
- Reports не меняет Warehouse facts и читает утверждённые события/проекции.
- Wiki развивается отдельными changesets и не блокирует Warehouse.
- Рабочая БД, raw sources, candidate DB, test DB и release artifacts имеют
  разные lifecycle и не подменяют друг друга.

## Организационные роли

- Владелец продукта утверждает бизнес-логику и опасные изменения.
- Codex исследует repository, реализует, тестирует и готовит evidence.
- Claude выполняет focused independent review критических инвариантов.
- Архитектурное review не заменяет работающий код и executable tests.
