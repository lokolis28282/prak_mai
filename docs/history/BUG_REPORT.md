## Найденные замечания

### BUG-001

Приоритет: P3

Файл: окружение проверки

Функция: запуск тестов через `python3 -m pytest -q`

Причина: в текущем Python-окружении не установлен пакет `pytest`.

Как воспроизвести:

```bash
python3 -m pytest -q
```

Фактический результат: `No module named pytest`.

Как исправить: либо установить `pytest` в QA-окружение, либо официально использовать `python3 -m unittest -v tests.test_warehouse` как поддерживаемую команду регресса. В текущей проверке `unittest` пройден полностью.

### BUG-002

Приоритет: P3

Файл: `data/warehouse.db`

Функция: админский вход в текущую рабочую БД

Причина: пароль пользователя `lokolis` в текущей БД отличается от дефолтного `lokolis`.

Как воспроизвести:

```bash
python3 - <<'PY'
from inventory.service import WarehouseService
s = WarehouseService("data/warehouse.db")
s.authenticate("lokolis", "lokolis")
PY
```

Фактический результат: `WarehouseError: Неверный email или пароль`.

Как исправить: если это продовая БД, ничего не менять. Если это должна быть тестовая БД с дефолтным доступом, сбросить пароль админа штатной процедурой или пересоздать тестовую БД.
